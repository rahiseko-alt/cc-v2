# 学習ロードマップ（docs）

新リポ作成を **スタート**、ユーザーが使えるアプリが動いている状態を **ゴール** とした、
アプリ制作の全体像を「育てて使う」学習地図。

- **図（本体）**: [`roadmap.html`](./roadmap.html) — 折りたたみツリー。解説は入れない、図だけ。
- **README（この文書）**: 使い方・カード表記ルール・育てログ。

## 使い方

- 一度に細かくしない。掘りたい枝を「**◯番を3つに割って**」と依頼 → その枝だけ3分割して追記する。
- どの AI ツール（Claude Code / Codex / Cursor / Antigravity）でも、`roadmap.html` を地図として参照・更新する。

## カード表記ルール

項目名は書かず、**値だけを順に並べる**。

```
番号  テーマ  意味  詳細  完了日
```

例:

```
1-1-1  ヒアリング  制約を洗い出す  何が動かせない？ クラウド/言語指定・既存連携・顧客/チームの縛り・予算/納期  2026-07-20
```

- **色 = 状態**: 未完了 → 無色 / 完了 → 青
- **完了日**: 完了したら日付を入れる。未完了は空欄。

## 育てログ

- 2026-07-20: 全体を3分割（準備／作る／出す）。1 を 1-1〜1-3、1-1 を 1-1-1〜1-1-3 に分割。
- 2026-07-20: 図（roadmap.html）と README を分離。カードを1行テキスト表記（番号/テーマ/意味/詳細/完了日）に統一。
