# 学習ロードマップ（docs）

新リポ作成を **スタート**、ユーザーが使えるアプリが動いている状態を **ゴール** とした、
アプリ制作の全体像を「育てて使う」学習地図。

- **図（本体）**: [`roadmap.html`](./roadmap.html) — 折りたたみツリー。解説は入れない、図だけ。
- **README（この文書）**: 目的・使い方・カード表記ルール・**作図の作法**・育てログ。

---

## この図の目的・意図

- **誰のため**: 非エンジニアが、アプリ制作の全体像を **体系的に** 掴むため。
- **なぜ作る**: 行き当たりばったりの経験に **名前と位置** を与え、「今どこを、なぜやっているか」を常に地図上で確認できるようにする。
- **育て方の思想**: 一度に細かくしない。知りたい枝だけ「**◯番を3つに割って**」で3分割し、その枝だけ深掘りする（3分解 × 3分解）。
- **この型（折りたたみツリー）を選んだ理由**: 最小の要素で以下を同時に満たすから。
  - 順番 → **番号**（1 / 1-1 / 1-1-1）
  - 階層 → **ツリーのインデントと縦線**
  - 進捗 → **カードの色**（青＝完了 / 無色＝未完了）
  - 俯瞰 → **折りたたみ**（普段は閉じ、見たい枝だけ開く）
  - ※「依存（並列・ブロッカー）」の表現は複雑になるため、現時点では **意図的に保留**。
- **運用**: どの AI ツール（Claude Code / Codex / Cursor / Antigravity）でも、この `roadmap.html` を地図として参照・更新する。

## カード表記ルール

項目名は書かず、**値だけを順に並べる**。改行は **意味の後、詳細の前**。

```
番号  テーマ  （意味）
詳細  完了日
```

- **番号** = 順番と階層（例: `1-1-1`）
- **テーマ = 専門用語**（例: ヒアリング / スタック選定 / スキャフォールド）
- **意味 = 非エンジニア用語**。かっこ（）で囲む（例: （動かせない条件を洗い出す））
- **詳細** = 具体的な中身。2行目に置く。
- **完了日** = 完了したら日付（`YYYY-MM-DD`）。未完了は空欄。

例:

```
1-1-1  ヒアリング  （動かせない条件を洗い出す）
何が動かせない？ クラウド/言語指定・既存連携・顧客/チームの縛り・予算/納期  2026-07-20
```

## 作図の作法（デザイン仕様）

**他の図に流用してもぶれないための共通仕様。** 色は必ずトークン（CSS変数）経由で使い、値を直書きしない。

### カラートークン（ライト / ダーク）

| トークン | 用途 | ライト | ダーク |
|---|---|---|---|
| `--paper` | ページ背景 | `#fbfbfd` | `#0f1117` |
| `--panel` | カード地・ボタン地 | `#ffffff` | `#161922` |
| `--ink` | 本文・見出し | `#1b1e26` | `#e7e9f0` |
| `--muted` | 番号・詳細・補助テキスト | `#5a606f` | `#9298aa` |
| `--line` | カード境界線・ツリー縦線 | `#a9afc0` | `#4a5266` |
| `--blue-fill` | 完了カードの地 | `#e3ebfb` | `#1a2340` |
| `--blue-edge` | 完了カードの境界 | `#7f9be6` | `#4d63a6` |
| `--blue-num` | 完了カードの番号色 | `#2c50c4` | `#a6b6f7` |

- **状態＝色**: 未完了 → 無色（`--panel` 地／`--line` 境界）、完了 → 青（`--blue-fill` 地／`--blue-edge` 境界／番号 `--blue-num`）。
- テーマ切替は `prefers-color-scheme` ＋ ルートの `data-theme` 上書きの両対応。

### タイポグラフィ

| 要素 | フォント | サイズ | 太さ |
|---|---|---|---|
| 基本（本文・テーマ・意味） | `ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, "Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif` | 14px | 400（テーマのみ 600）|
| 番号・完了日 | `ui-monospace, SFMono-Regular, Menlo, monospace`（`tabular-nums`）| 番号=14px / 完了日=12px | 番号=700 |
| ツールボタン | 基本と同じ | 12px | 400 |
| キャレット（▸）| 基本と同じ | 11px | 400 |

- 行間（line-height）: 1.45
- 番号は等幅＋`tabular-nums` で桁を揃える。

### 線・角丸・余白（余白は最小）

| 項目 | 値 |
|---|---|
| カード境界線 | 1.5px `--line`（完了は `--blue-edge`）|
| ツリー縦線（子のインデント左線）| 2px `--line` |
| ボタン境界線 | 1px `--line` |
| カード角丸 | 7px |
| ボタン角丸 | 6px |
| カード内パディング | 4px 8px |
| カード内 gap | 6px |
| ノード上下マージン | 4px |
| 子インデント | margin-left 15px / padding-left 13px |
| 詳細行（2行目）上マージン | 2px |
| 番号の右マージン | 8px ／ テーマの右マージン 7px ／ 完了日の左マージン 8px |

### 構造・インタラクション

- **1カード = 2行**。1行目＝番号／テーマ／（意味）、2行目＝詳細／完了日。
- **折りたたみ**: `<details>`/`<summary>` で実装。子を持つカードはキャレット `▸`（開くと90°回転）。葉（子なし）はキャレット非表示。
- 画面上部に「開く／閉じる」で全展開・全折りたたみ。
- 図のページには **解説・凡例・脚注を置かない**（説明はこの README に集約）。
