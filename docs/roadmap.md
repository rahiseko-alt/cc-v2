# 学習ロードマップ（アプリ制作の全体地図）

新リポ作成を **スタート**、ユーザーが実際に使えるアプリが動いている状態を **ゴール** とした、
アプリ制作の全体像。行き当たりばったりを卒業し、体系的に理解していくための「生きた地図」。

## この地図の使い方（育て方の規約）

- **一度に細かくしない。** まず全体を 3 分割し、知りたい枝だけ深掘りする。
- **3 分解 × 3 分解で育てる。** 掘りたいときは「**◯番を3つに割って**」と依頼する。
  依頼された枝だけを 3 つに分解して、このファイルに追記していく。
- どの AI ツール（Claude Code / Codex / Cursor / Antigravity）でも、このファイルを地図として参照・更新する。

## 全体地図（まず3分割）

```mermaid
mindmap
  root((新リポからアプリ完成まで))
    1 準備する ★いまここ
      1-1 決める・スタック選定
      1-2 土台をつくる・リポジトリと骨組み
      1-3 整える・ルールと環境
    2 作る
      コードで中身をつくる
      次は3分解
    3 出す
      公開して運用する
      次は3分解
```

## 3つのフェーズ（ひとことで）

| # | フェーズ | ひとことで | 中身をざっくり |
|---|---|---|---|
| **1** | 準備する | 何で作るか決めて、土台を用意する | 新リポ作成・スタック選定・フォルダ/設定の土台 |
| **2** | 作る | コードを書いて、動く状態にする | 画面・処理・データ、そして品質チェック |
| **3** | 出す | 公開して、使える・保てる状態にする | デプロイ・運用・直し続ける仕組み |

### 1. 準備する の3分解

| # | サブフェーズ | ひとことで | 中身 | cc-v2 でやったこと |
|---|---|---|---|---|
| **1-1** | 決める（スタック選定）| 何で作るか勝手に仮定せず決める | ヒアリング→リサーチ→提案→承認。クラウド/言語/FW/DB/認証… | 「決め方フロー」自体をデフォルト化 |
| **1-2** | 土台をつくる（器）| 決めた構成の「入れ物」を用意する | 新リポ・フォルダ構成・パッケージ管理・設定ファイル | pnpm モノレポ（apps/web・packages/ui・tsconfig.base） |
| **1-3** | 整える（ルールと環境）| どこでも同じに動き、AIも迷わない状態に | AGENTS.md・秘密情報の扱い・バージョン固定・.gitignore | AGENTS.md 統一/転送・.gitignore 整備 |

順番: **1-1 決める → 1-2 土台をつくる → 1-3 整える**
