学習ロードマップ — アプリ制作の全体地図

新リポ作成を スタート、ユーザーが使えるアプリが動いている状態を ゴール とした全体像。カードを開いて深掘りし、3分割で育てていく。

済（青） 未（無色）
▸1 準備する何で作るか決めて、土台を用意する
▸1-1 決める（スタック選定）何で作るかを、勝手に仮定せず決める
▸1-1-1 制約を洗い出す（ヒアリング）何が動かせない？ クラウド/言語指定・既存連携・顧客/チームの縛り・予算/納期
▸1-1-2 選択肢を調べる（リサーチ）制約の中で、その年のベストと候補ごとのトレードオフ（費用/相性/将来性）
▸1-1-3 決めて確定する（提案→承認）1案＋根拠＋代替1つ → 明示OK → 決定を AGENTS.md に記録
▸1-2 土台をつくる決めた構成の「入れ物」を用意する。新リポ・フォルダ構成・パッケージ管理・設定ファイル
▸1-3 整えるどこでも同じに動き、AIも迷わない状態に。AGENTS.md・秘密情報の扱い・バージョン固定・.gitignore
▸2 作るコードを書いて、動く状態にする。画面・処理・データ、そして品質チェック
▸2-? 未展開「2番を3つに割って」で分割していく
▸3 出す公開して、使える・保てる状態にする。デプロイ・運用・直し続ける仕組み
▸3-? 未展開「3番を3つに割って」で分割していく

この地図の使い方

  • 一度に細かくしない。掘りたい枝を「◯番を3つに割って」と依頼 → その枝だけ3分割して追記する。
  • 色＝状態。青＝済／無色＝未。
  • どの AI ツール（Claude Code / Codex / Cursor / Antigravity）でも、この docs/roadmap.html を地図として参照・更新する。
