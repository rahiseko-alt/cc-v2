この図の目的・意図

  • 誰のため: 非エンジニアが、アプリ制作の全体像を体系的に掴むため。
  • なぜ作る: 行き当たりばったりの経験に「名前と位置」を与え、今どこを・なぜやっているかを地図で確認できるようにする。
  • この型（折りたたみツリー）を選んだ理由: 最小要素で 順番＝番号 / 階層＝ツリー / 進捗＝色 / 俯瞰＝折りたたみ を同時に満たすから。依存（並列・ブロッカー）の表現は複雑になるため現時点は意図的に保留。
  • 運用: どの AI ツール（Claude Code / Codex / Cursor / Antigravity）でも、この図を地図として参照・更新する。

割り方の規則

目的で割る。コミット1個サイズになったら止める。

  • 粒度＝コミット単位（1カード＝1つの目的）。数は固定しない（2個でも6個でもよい）。
  • 完了の記録: 葉が終わったら＝コミット。カードを青にし、完了日＝コミット日を入れる。
  • 掘るときは「◯番を割って」と依頼 → その枝だけ、上記規則で分割・追記する。

カード表記ルール

項目名は書かず、値だけを順に並べる。改行は意味の後、詳細の前。

番号  テーマ  （意味）
詳細  完了日
  • テーマ = 専門用語（例: ヒアリング / スタック選定 / スキャフォールド）
  • 意味 = 非エンジニア用語。かっこ（）で囲む（例: （動かせない条件を洗い出す））
  • 詳細 = 具体的な中身（2行目）。完了日 = YYYY-MM-DD、未完了は空欄。

他の図に流用してもぶれないための共通仕様。色は必ずトークン経由で使い、値を直書きしない。
